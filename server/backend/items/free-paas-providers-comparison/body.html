Notes:

Almost all of the platforms can scale freely. Most of the platform scale painlessly. It works by adding a new CPU core, running a copy of the application, with a load balancing server that automatically distributes the request between the application instances.

Almost all platforms have a terrible initial response time. In the free tier, your application is swaped out of memory after some n minutes of inactivity. That means that the first user in every n-minute time interval needs to wait for the application to be loaded into memory. If the PaaS is runnins SSD's, i've measured the delay to be around 1.5s. The worst performer is Microsoft Azure, where I measure an average of 4s delay. The best performer goes to IBM's Bluemix with 0s delay. Bluemix is running on Cloud Foundry, which seems to not swap out processes at all.

In rare cases (Azure) applications don't allow for own domains.
